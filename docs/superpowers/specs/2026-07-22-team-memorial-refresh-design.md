# 球队纪念站整体视觉打磨设计

## 目标

将现有静态球队网站打磨成以真实照片、真实文字和赛季记录为主角的球队纪念册。网站继续部署在 GitHub Pages，不增加后端依赖、第三方账号或运行成本。

## 视觉方向

采用“球场手记”作为主轴：克制、纪实、有阅读节奏。保留绿色作为球队识别色，使用近白纸张底色、深绿正文和有限的暖色强调。照片承担情绪，文字和数据承担说明；避免商业球队官网式的大量口号、厚重渐变和装饰性卡片。

## 首页

- 保留三张真实合照和手机端左右滑动。
- 每张首页图使用“赛季扉页”排版：左上为来源或章节，小标题；主标题保持横向、清晰、不过度覆盖人物；行动按钮保持在照片下部安全区域。
- 底部的建队年份、球员档案和主场信息形成统一的轻量坐标带，不与人物主体重叠。
- 导航在顶部保持简洁，滚动后采用低对比半透明底，当前页状态清楚但不抢视觉重心。
- 幻灯片只使用缓慢缩放和淡入，切换不使用强烈位移；尊重 `prefers-reduced-motion`。

## 沉浸式首发阵容

- 保持现有球场比例和 4-3-3 站位；球场是主画面，不能因右侧信息而缩小。
- 右侧绿色信息板升级为“阵线档案卡”：球队名和赛季为小型页眉，阵线名为大标题，阵型说明、当组首发名单与进度条形成固定层级。
- 电脑端继续以页面滚动依次聚焦前锋、中场、后卫、门将；右侧档案卡同步更新。
- 手机上页面上下滚动不被阵容模块劫持；在球场或绿色档案卡上左右滑动可切换阵线，前锋向左可循环到门将，向右进入中场。底部标签仍可直接点击切换。
- 聚焦阵线的队员卡清晰放大，其他位置降为背景层但仍可辨认；点击队员保持打开档案弹窗的现有行为。

## 赛事与荣誉

- 年份和杯赛索引维持“年份 → 杯赛”的两级逻辑，默认展示 2025 十一人制纺大杯。
- 当前赛事的概览以档案封面形式呈现：赛事名、赛季、简短的真实背景说明和比赛数量。
- 单场比赛以纵向时间线记录，日期、对手、比分和真实赛后文字各自有稳定位置。
- 荣誉区域从均质卡片改为少量有编号、有年份、可扫描的时刻条目；不虚构未发生的成绩。

## 影像墙

- 保持真实照片和现有分类，不改变照片故事归属。
- 照片排版继续采用瀑布流，但标签、点赞和文字信息采用更轻的叠层；文字对比度必须足够，不能遮挡照片主体。
- 影像卡悬停与触摸反馈只做轻微缩放和暗角变化；点赞按钮独立响应，不触发打开照片。
- 图片继续使用 WebP 和懒加载，打开详情时展示照片标题、分类和故事。

## 全队档案与弹窗

- 随机展示的三张队员卡改为年鉴式封面，号码、姓名、位置和踢球风格为正面主信息；保留随机提示。
- 弹窗维持真实数据、评分、队友评价弹幕、位置与踢球风格；删除视觉上无意义的空白或重复装饰。
- 不补写没有来源的评价、比赛数据或荣誉。

## 移动端与性能

- 手机优先检查导航、首页标题、照片裁切、阵容信息卡、影像墙和模态框；所有文字应在容器内换行或裁切，不能覆盖人物和控件。
- 移动端点击目标不小于 44px，横向手势只在阵容和首页图中生效，其余区域保持自然竖向滚动。
- 图片请求保持策略：第一张首页图与首发头像优先，后续首页图预取，影像墙按可视区域加载。
- 不新增大型依赖；继续使用现有 HTML、CSS 和原生 JavaScript。

## 验收标准

- 桌面和手机截图中，首页标题、信息带和导航不遮挡合照主体。
- 桌面下滑可依次切换四条阵线；手机上下滚动连贯，左右滑动可循环切线。
- 影像墙可正常筛选、点赞、打开详情，且离屏图片不会在首次进入首页时加载。
- 赛事索引、队员弹窗、随机队员展示、现有真实照片和数据均可正常工作。
- 在 Chrome 桌面与手机模拟视口完成视觉检查；JavaScript 通过语法检查；改动提交并推送到 `main`。
