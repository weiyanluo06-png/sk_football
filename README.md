# 生康足球队纪念站

武汉纺织大学生物医学工程与健康学院足球队的记录网站。这里保存建队、训练、比赛、队员与球队生活中的真实照片和资料。

**线上访问：** [weiyanluo06-png.github.io/sk_football](https://weiyanluo06-png.github.io/sk_football/)

## 这支球队

生康足球队于 2025 年开始组建。最初因为不能再以联院形式参赛，学院人数又少，队伍一度连能否成形都不确定。后来有人持续拉人报名、订队服，队员们一起把人数凑齐；第一次排阵容时，大家在食堂讨论出了最初的 4-2-3-1 站位。

球队的第一段正式比赛是十一人制纺大杯。首战在雨中以 2:4 对阵管理学院，虽然没有赢球，但球队没有被打散，打出了自己的风采。之后在首届八人制新生杯中，球队最终取得第七名。

材料学院一直是生康足球队的兄弟学院。从球队还没有成形，到成立初期的场边指导和加油，材料学院的学长给过球队很多帮助。这份关系也会继续传给之后加入的新队员。

## 网站内容

- **首页轮播**：球队合照、建队故事与材料学院的兄弟情谊。
- **新赛季招新**：首页导航和首屏入口可直达招新区，扫码加入 QQ 群；群号为 `913800697`。
- **赛事档案**：按年份和杯赛索引浏览比赛。当前默认展示 `2025 · 十一人制纺大杯`。
- **沉浸式首发阵容**：固定 4-3-3 首发。桌面端滚动依次聚焦前锋、中场、后卫和门将；手机端左右滑动球场或绿色信息卡切换阵线。
- **队员档案**：展示真实球员资料、头像、位置、评分、踢球风格和队友评价。
- **影像墙**：雨天训练、比赛围圈、球队合照、毕业与场外日常；支持点赞和照片详情查看。
- **队史节点**：记录建队、十一人制首战、首届新生杯第七名与材料学院的帮助。

## 赛事归档

学校每学年有三段学院足球赛事：

1. 上学期：十一人制纺大杯
2. 上学期：八人制新生杯
3. 下学期：八人制纺大杯

网站先按年份索引，再按杯赛切换。目前已收录 `2025 · 十一人制纺大杯`、`2025 · 八人制新生杯` 和 `2026 · 八人制纺大杯` 的信息。

## 更新内容

### 用 Excel 同步球队数据

日后只需要编辑 [生康足球队数据源.xlsx](data/%E7%94%9F%E5%BA%B7%E8%B6%B3%E7%90%83%E9%98%9F%E6%95%B0%E6%8D%AE%E6%BA%90.xlsx) 里的三张表，再把更新后的文件交给维护者同步即可：

- `球员数据`：号码、位置、出场、进球、助攻、MVP、零封、踢球风格与评价。
- `赛事索引`：赛季的年份、赛事名称与简介。
- `赛程`：每场比赛的日期、对手、比分和比赛信息。

评分会根据表内数据自动计算：基础分 6 分，再综合出场、进球、助攻、MVP 和零封，最终限制在 6.0 至 9.5 分。代表数据也会随进球、助攻和出场自动更新。

### 更新球员资料或首发

编辑 [js/team-data.js](js/team-data.js)：

- `players`：姓名、号码、位置、数据、评价、踢球风格和照片路径。
- `startingLineup`：固定首发的位置与队员。修改对应位置的 `playerId` 即可更换首发。
- `competitions` 与 `matches`：赛事索引和比赛记录。
- `honors`：队史节点。

球员照片放在 `assets/players/`。推荐使用 WebP，竖版头像在阵容展示中效果最好。

### 更新首页、影像墙或招新二维码

- 首页轮播：编辑 [index.html](index.html) 中 `.hero__slide` 的图片路径和文案。
- 影像墙：编辑 [js/gallery-data.js](js/gallery-data.js) 中的 `image`、`title`、`category` 和 `story`。
- 招新二维码：替换 `assets/photos/recruit-qq-group.jpg`，保留清晰原图，避免压缩或在二维码上叠加文字。

首页和影像墙已经使用 WebP 与延迟加载来缩短手机端首次加载时间。新增照片时，建议同步准备 WebP 版本，并使用新文件名避免旧缓存。

### 更新招新文案

招新板块位于 [index.html](index.html) 的 `#recruit` 区域。可以直接修改标题、介绍、QQ群号和二维码图片路径。

```powershell
python -m http.server 8061 --bind 127.0.0.1
```

然后访问 [http://127.0.0.1:8061/](http://127.0.0.1:8061/)。

## 发布到 GitHub Pages

网站使用 GitHub Actions 自动发布。将修改提交并推送到 `main` 分支后，GitHub Pages 会自动更新：

```powershell
git add README.md index.html css/style.css js/team-data.js js/gallery-data.js assets
git commit -m "更新球队资料"
git push origin main
```

推送后可在仓库的 `Actions` 页面查看 `Deploy GitHub Pages` 是否部署成功。通常等待一两分钟，再刷新线上页面即可看到更新。
